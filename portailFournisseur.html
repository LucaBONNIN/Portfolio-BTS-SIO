C# ASP.NET Core Entity Framework HTML / CSS JavaScript SQL Server ✓ En production

📋Contexte du projet

Spie Batignolles est un groupe français majeur du BTP intervenant dans la construction, les travaux publics et l'énergie. Ses chantiers font intervenir de nombreux sous-traitants et agences d'intérim qui doivent régulièrement soumettre leurs factures au service comptabilité.

Avant ce projet, ce processus était géré manuellement par email, sans format standardisé, ce qui générait des erreurs, des oublis de pièces jointes et une perte de temps importante. L'objectif était de créer un portail web unifié permettant aux fournisseurs de déposer leurs factures en ligne avec tous les justificatifs requis, les données étant automatiquement transmises par email et enregistrées en base de données.

Projet développé en 1 mois dans le cadre de l'alternance, à partir d'une coquille initiale fournie par l'entreprise. L'ensemble de la base de données et du code frontend / backend ont été construits par l'équipe.

✨Fonctionnalités

📋

Formulaire dynamique

Le formulaire adapte ses champs en temps réel selon le type de facture sélectionné via JavaScript, sans rechargement de page.

  • 3 types : Sous Traitance, Intérim, Autres
  • Champs conditionnels (DSK, préfacture…)
  • Validation des champs obligatoires
📎

Upload de pièces jointes

Le fournisseur peut joindre sa facture et ses documents complémentaires directement depuis le formulaire.

  • Sélection multi-fichiers
  • Transmission par email avec les données
📧

Notification par email

À chaque soumission, un email formaté reprenant toutes les données saisies est envoyé automatiquement au service comptabilité concerné.

  • Adresse destinataire selon le type de facture
  • Corps du mail structuré en tableau
  • Pièces jointes incluses
🗃️

Persistance en base de données

Chaque dépôt de facture est enregistré en base SQL Server via Entity Framework, permettant un historique complet des soumissions.

  • Stockage de tous les champs du formulaire
  • Clés étrangères vers Societes et Type_Factures
🏢

Multi-sociétés

Le fournisseur sélectionne la société Spie Batignolles concernée depuis une liste déroulante alimentée dynamiquement depuis la base de données.

  • Table dbo.Societes administrable
  • Correspondance automatique email / type
🎨

Charte graphique Spie Batignolles

Interface entièrement aux couleurs de l'entreprise : logo officiel, couleur ambre, typographie sobre adaptée à un usage professionnel.

  • Logo Spie Batignolles intégré
  • Palette officielle orange / blanc

🗄️Modèle de données

La base de données portail_fournisseur (SQL Server) contient trois tables principales gérées via Entity Framework Core. Les migrations EF assurent le versionnage du schéma.

dbo.Societes
PK ID int, not null
LIBELLE nvarchar, not null
dbo.Type_Factures
PK ID int, not null
LIBELLE nvarchar, not null
EMAIL nvarchar, not null
dbo.Factures
PK ID int, not null
SIRET nvarchar, not null
CAE nvarchar, not null
FACTURE nvarchar, not null
FK ID_TYPE_FACTURE int
COMMANDE nvarchar, null
CONTRAT nvarchar, null
CONTRAT_DSK nvarchar, null
INTERLOCUTEUR nvarchar, not null
MAIL nvarchar, null
MONTANT_TTC nvarchar, not null
PREFACTURE nvarchar, null
FK ID_SOCIETE int, null
→ Factures.ID_SOCIETE référence Societes.ID (FK_Factures_Societes)
→ Factures.ID_TYPE_FACTURE référence Type_Factures.ID (FK_Factures_Type_Factures)
Les champs null dans Factures sont conditionnels selon le type (DSK et préfacture uniquement pour Intérim, Contrat uniquement pour Sous Traitance, etc.)
